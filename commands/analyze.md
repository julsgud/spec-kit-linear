# Analyze

You are performing a cross-artifact consistency and coverage analysis on all spec-kit comments for a Linear issue.

## Input

The user provides a Linear issue URL or identifier as `{{ args }}`.

## Step 1: Parse the issue identifier

Extract the issue identifier from the input:
- `https://linear.app/{workspace}/issue/{ID}/{slug}` → use `{ID}`
- `https://linear.app/{workspace}/issue/{ID}` → use `{ID}`
- `BOT-140` (bare identifier) → use directly

If the input does not match any of these formats, stop with:
```
Error: Could not parse Linear issue from "{{ args }}".
Expected a Linear URL (https://linear.app/.../issue/BOT-140/...) or identifier (BOT-140).
```

## Step 2: Read the issue and all comments

Call `get_issue` with the extracted identifier.

Call `list_comments` and collect all spec-kit artifact comments by searching for sentinels:
- `<!-- speckit:specification:v* -->` — Specification
- `<!-- speckit:clarifications:v* -->` — Clarifications (optional)

For each artifact type, use the comment with the highest version number.

If no specification comment is found, stop with:
```
Error: No Specification found on {ID}.
Run /speckit.linear.specify {ID} first.
```

## Step 3: Check for existing analysis

Search the comments for existing analysis sentinels (`<!-- speckit:analysis:v* -->`). If previous versions exist, note the highest version number.

## Step 4: Perform cross-artifact analysis

Review all artifacts together (issue description, specification, clarifications if present) and assess:

### Coverage Assessment
How well does the specification cover the original issue description? Are all requirements from the description addressed? Rate coverage as Complete, Mostly Complete, Partial, or Insufficient.

### Consistency Check
Do the artifacts agree with each other? Flag any requirements in the specification that contradict the original description, or clarifications that reveal specification gaps.

### Risk Assessment
Identify technical risks, complexity hotspots, and areas most likely to cause implementation issues. Rate each as Low, Medium, or High risk.

### Completeness Gaps
What is still missing after the specification and any clarifications? What decisions remain unmade?

### Recommendations
Specific actions to improve the artifacts before moving to planning. These might include:
- Re-running `/speckit.linear.specify` with additional context
- Running `/speckit.linear.clarify` if not yet done
- Manual stakeholder input needed on specific questions

## Step 5: Post the analysis comment

Determine the version number:
- If no previous analysis comment exists → `v1`
- If previous versions exist → increment

Format the comment body as:

```markdown
<!-- speckit:analysis:v{N} -->
## Analysis

> Generated by spec-kit-linear | v{N} | {today's date}

### Coverage Assessment
{coverage assessment}

### Consistency Check
{consistency findings}

### Risk Assessment
{risk items with severity ratings}

### Completeness Gaps
{remaining gaps, or "None — ready for planning."}

### Recommendations
{actionable recommendations}
```

Call `save_comment` to post this as a new comment on the issue.

If a previous version exists, edit the previous version's comment to prepend `**(superseded by v{N})**` on the line immediately after the `## Analysis` header.

## Output

Summarize what was done:
- Issue: `{ID} — {title}`
- Artifacts reviewed: list which artifacts were found and their versions
- Analysis version: `v{N}`
- Overall readiness: Ready for planning / Needs attention
